# Speak Coach AI

Live Demo: https://speak-polish-ai.vercel.app/

Speak Coach AI は、発音練習をより実践的かつ継続しやすくするために開発した、AI ベースのスピーキング練習アプリです。音声ベースの発音フィードバック、自然な音声再生、入力中の英文リライト支援、簡易的な学習記録を、ひとつのフロントエンド中心の体験にまとめています。

## このアプリの特徴

- 文字起こし結果ではなく、実際の音声をもとに発音を評価する音声ファーストのコーチング
- AI 生成の TTS と部分リピート再生により、苦手なフレーズを集中的に練習できる設計
- 文中の選択範囲に対して、文法説明と自然な言い換え候補をその場で提示し、そのまま反映できる UI
- 直接の音声解析が不安定な場合でも、文字起こしベースの評価にフォールバックする堅牢なフィードバック基盤

## 概要

このプロジェクトは、技術的には複雑になりやすい AI 機能を、毎日使えるシンプルな練習体験に落とし込むことを目的に開発しました。ユーザーは練習文を入力または生成し、ネイティブらしい音声を聞き、自分の声を録音し、AI から発音フィードバックを受け取り、その結果を記録できます。

対象ユーザーは、英語を話す練習をしたい非ネイティブ学習者です。講師や高価なツールに頼らず、すぐに使えて、すぐに改善に繋がるフィードバックを得られる体験を目指しました。

## 開発した理由

スピーキング練習は、良いフィードバックをすぐにもらえないことが多く、継続のハードルが高いと感じています。そこで、次の流れを短い体験の中で完結させるプロダクトを作りたいと考えました。

- まず自然な音声を聞く
- その場で同じ文を話す
- どう聞こえたかのフィードバックを受ける
- 不自然な表現は文中で書き換える
- 練習結果を蓄積して継続しやすくする

## プロダクト構成

### 1. Practice Screen

メイン画面は、入力、再生、録音、振り返り、改善、という一連の学習ループを短く回せるように設計しています。

![Top screen](public/demo.gif)

ユーザーは次の操作を行えます。

1. 文を手入力する、または難易度別 (`Easy`, `Middle`, `High`) に生成する
2. TTS で自然な音声を再生する
3. 選択した単語やフレーズだけを部分再生する
4. 選択範囲に対して文法説明と言い換え候補を取得する
5. 自分の音声を録音し、AI による発音評価を受ける
6. 結果を保存し、ストリーク・XP・レベルの進捗を積み上げる

### 2. History Screen

過去の練習履歴を一覧で確認できます。

- 練習した文と修正文
- 発音スコアの履歴
- 日々の学習ログとしての振り返り

### 3. Progress Screen

継続のモチベーションを可視化する画面です。

- 現在のストリーク
- 総練習回数
- XP とレベル進行

### 4. Settings Screen

学習スタイルに合わせて環境を調整できます。

- テーマ切り替え: `Light`, `Dark`, `System`
- AI フィードバック言語: `English`, `Japanese`
- 表示設定やフィードバック設定の永続化

## 仕組み

1. ユーザーが練習したい文を入力、または生成します。
2. `/api/tts` にテキストを送り、自然な TTS 音声を生成します。
3. ブラウザの `MediaRecorder` を使ってユーザーの音声を録音します。
4. 録音音声と目標文を `/api/pronunciation-feedback` に送信します。
5. サーバー側ではまず文字起こしを行い、その後、音声そのものを使った発音解析を試みます。
6. 直接の音声解析に失敗した場合は、文字起こし結果と単語タイミングを使った評価にフォールバックします。
7. テキストの選択範囲は `/api/text-feedback` に送られ、文法説明と言い換え候補を返します。
8. 練習結果はローカルに保存され、ストリーク、XP、レベル、履歴の表示に使われます。

内部では、音声再生に TTS、文字起こしに ASR、発音評価や文法フィードバック生成に LLM を使い分けています。

## 実装で重視した点

このプロジェクトでは、単に AI を呼び出すことよりも、複雑な処理を違和感のない体験にまとめることを重視しました。

- 録音、再生、AI 応答、UI 遷移が並行する中での非同期状態管理
- デスクトップとモバイルの両方で破綻しにくい音声操作 UX
- OpenAI 連携を担う軽量な API レイヤーの設計
- AI の生データを安定したユーザー向けフィードバック構造に正規化する処理
- 音声解析が遅い、または不安定な場合でも体験を崩しにくい設計

## 課題と設計判断

### 1. 音声解析の信頼性

音声そのものを使った解析は、文字ベースより質の高いフィードバックを返せる一方で、不安定になりやすい側面があります。そのため、まず録音音声を文字起こしし、その結果と単語タイミングを使った評価に切り替えられるフォールバック経路を用意しました。

### 2. フロントエンドの状態管理

このアプリでは、ひとつの画面で次のような非同期処理が同時に関わります。

- 音声録音
- TTS 再生
- 選択範囲の部分再生
- テキストフィードバック取得
- 発音フィードバック取得
- 保存後の進捗更新

そのため、録音中、再生中、ローディング中、結果表示中といった状態が衝突しないよう、UI の整合性を意識して設計しました。

### 3. AI 機能の UX への落とし込み

AI 機能は増やすだけでは使いにくくなりがちです。このアプリでは各機能を、ユーザーが理解しやすい具体的な操作に落とし込みました。

- `Listen`
- `Listen Selection`
- `Analyze text`
- `Record`
- `Save`

その結果、複数の AI 機能を並べるのではなく、学習フローの中で自然に使える構成にしています。

## 補足

- 発音スコアは AI による推定であり、無関係な音声や非英語音声に高得点が出にくいようガードを入れています
- 音素単位の厳密な研究用途評価ツールではありません
- ブラウザ対応は `MediaRecorder` とマイク権限に依存します
- ストリーク、XP、レベル、履歴はブラウザの `localStorage` に保存しています

## セットアップ

Node.js `20+` が必要です。

```bash
node -v
```

`.env.local` を作成します。

```bash
OPENAI_API_KEY=your_api_key

# Optional
OPENAI_AUDIO_ANALYSIS_MODEL=gpt-4o-audio-preview
OPENAI_MODEL=gpt-4.1-mini
OPENAI_TRANSCRIBE_MODEL=gpt-4o-mini-transcribe
OPENAI_TEXT_FEEDBACK_MODEL=gpt-4.1-mini
OPENAI_TTS_MODEL=gpt-4o-mini-tts
OPENAI_TTS_VOICE=alloy
```

## 起動方法

```bash
npm install
npm run dev
```

ブラウザで `http://localhost:3000` を開いてください。
